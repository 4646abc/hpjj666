# PUBG Mobile 物品代码生成器 — AI 维护手册

## 项目概述
单文件离线 HTML 工具，用于生成 PUBG Mobile（com.tencent.tmgp.pubgmhd，UID:2965）的物品替换代码。
所有数据内嵌于 HTML，完全离线可用，无需服务器。

---

## 一、数据来源与结构

### 数据文件格式（.h 文件）
```
{ID}--{关联ID}--[名称]--[偏移]
```
- **ID**：物品唯一标识（4~9位数字）
- **关联ID**：若等于自身 = 真实可用皮肤；若为 `0` = 占位符/无效条目
- **名称**：物品显示名称（可能被截断，如 `M416-天马神君(弹匣` 无闭合括号）
- **偏移**：十六进制偏移值

### 数据处理流程
1. Python 脚本解析 .h 文件 → 生成 JSON
2. gzip 压缩 → base64 编码 → 嵌入 HTML 变量 `DB`
3. 前端用浏览器内置 `DecompressionStream` API 解压（替代 pako.js，无需 CDN）

> ⚠️ **重要**：`DecompressionStream` 是异步 API，`loadDB()` 必须用 `async/await` 或 `.then()` 实现，不能同步调用。

## 二、页面功能结构

### Tab 1：武器皮肤
点击皮肤 → 调用 `genWCodes()` → 按固定顺序生成所有配件代码。

**代码生成顺序**（严格固定）：
1. 挂件 `武器ID-挂件-挂件ID`
2. 击打特效 `武器ID-击打特效-特效ID`
3. 检视动画 `武器ID-检视-检视ID`
4. 主体 `武器ID-武器ID-皮肤ID`
5. 机械瞄、枪口、弹匣、枪托
6. 补偿器(201009)、消焰器(201010)、消音器(201011)
7. 直角握(202001)、垂直握(202002)、轻型握(202004)、半截握(202005)、拇指握(202006)、激光(202007)
8. 扩容弹(204011)、快速弹(204012)、快扩弹(204013)
9. 战术枪托(205002)
10. 红点(203001)→全息(203002)→2倍(203003)→3倍(203014)→4倍(203004)→6倍(203015)→侧面(203018)
11. 耍酷动作(1001xxx)、换弹效果(1002xxx)、开火效果(1003xxx)

**特效 ID 前缀与最大长度**（maxLen 防误匹配）：
| 类型 | 前缀 | 最大ID长度 |
|------|------|----------|
| 枪械挂件 | 801 | 6 |
| 击打特效 | 905 | 6 |
| 检视动画 | 1004 | 7 |
| 耍酷动作 | 1001 | 7 |
| 换弹效果 | 1002 | 7 |
| 开火效果 | 1003 | 7 |

**附件匹配规则**：
- 附件名称必须同时包含「武器名」AND「皮肤名」才算匹配
- 武器名从 `WEAPONS` 字典取（处理 SCAR-L、AKS-74U 等含横线武器）
- 皮肤名 = bName 去掉「武器名-」前缀后的剩余部分

**投掷物（602xxx）特殊规则**：
只生成一行：`武器ID-武器ID-手持特效ID`
- 烟雾弹(602002) → 手持前缀 923xxx
- 燃烧瓶(602003) → 手持前缀 933xxx
- 手榴弹(602004) → 手持前缀 920xxx
- 铝热弹(602075) → 手持前缀 920xxx

**代码注释格式**（复制时附带）：
```
#m416击打特效改天马神君
101004-击打特效-905545
#m416改天马神君
101004-101004-101004146
```

**无专属特效时**：
- 显示「浏览挂件/击打特效/检视动画」按钮
- 点击展开对应 ID 前缀的全部可选列表
- 选中后计入「复制全部代码」

### Tab 2：衣服（局内美化 & 全局换装）

#### 局内美化 — 三种模式

**① 普通衣服**：14个部位格，输出 `部位 = ID`

**② 套装模式**：
- 浏览区显示 **真实套装**（413xxx/423xxx，`关联ID=自身ID`，非0）
- 自动填入 脸部 = `400985`（奥特曼专属头发三角形）
- 选衣服后自动匹配同系列三级背包(501003xxx)和三级头盔(502003xxx)

**③ 红装模式**：
- 浏览区显示 **19套真实红装**（三个一组：413/423xxx，`关联≠0`）
- 自动填入：
  - 脸部 = `400985`
  - 面罩 = 冲刺特效(66040xxx)
  - 眼罩 = 跳跃特效(66090xxx)
  - 帽子 = 治疗特效(66100xxx)
  - 头发 = 翻越特效(66110xxx)
  - 胸甲 = 同衣服 ID
  - 背包 = 同系列三级背包(501003xxx)
  - 头盔 = 同系列三级头盔(502003xxx)
- 无专属特效时默认填木乃伊(66040003/66090003/66100003/66110003)

> ⚠️ **关键规则**：430xxx 全部是 `关联=0` 的占位符，不是真实皮肤 ID，不能使用！

#### 全局换装
- 「改前: 手动/自动」切换：自动模式点选皮肤自动填 关联ID→改前、皮肤ID→改后
- 生成：改前-改后 + 回改 改后-改前
- 一键复制两行

### Tab 3：完美枪皮
从数据库筛出同时有「专属挂件+击打特效+检视动画」三项全的枪皮（约21套）。
点击「生成全套代码」→ 走完整 genWCodes 流程。

### Tab 4：皮肤分类浏览
按武器/衣服分类展示，点击展开代码。

### Tab 5：ID查询
全文搜索，支持按ID/名称/偏移过滤，点击卡片复制ID。

### 配置存档库
- 生成代码后「存入配置库」保存到内存数组（最多50套）
- 支持查看/复制/删除，Tab 显示角标数量
- 刷新页面后清空（内存存储）

## 四、已修复的重要 Bug

### Bug 1：SCAR-L/AKS-74U 附件搜不到
**原因**：武器名 `SCAR-L` 被 `-` 分割后 `wn='SCAR'`，皮肤名 `sk='L-魔童闹海 哪吒'`，搜索失败。
**修复**：从 `WEAPONS` 字典取正确武器名，再用 `bName.slice(wn.length+1)` 提取皮肤名。

### Bug 2：S12K/霰弹枪/轻机枪搜不到
**原因**：搜索过滤器只允许 `id[2]` 在 1/2/3（步枪/冲锋/狙击），霰弹枪是4，轻机枪是5。
**修复**：过滤器增加 `id[2]==='4'` 和 `id[2]==='5'`，并加入 602xxx（投掷物）和 107/108xxx（弓弩近战）。

### Bug 3：击打特效/检视动画搜不到（哪吒）
**原因**：同 Bug 1，武器名提取错误导致皮肤名关键词错误。修复同 Bug 1。

### Bug 4：DecompressionStream 离线无数据
**原因**：之前用 CDN 加载 pako.js，本地打开时无网络导致解压库加载失败。
**修复**：换用浏览器内置 `DecompressionStream`（Chrome 80+/Android 10+ 支持），完全无需网络。

### Bug 5：filterLRes is not defined
**原因**：过滤按钮用 `onclick="filterLRes()"` 内联属性，在 WebView 的 CSP 限制下找不到函数。
**修复**：改用 `addEventListener('click', filterLRes)` 动态绑定。

### Bug 6：投掷物代码格式错误
**原因**：生成了 `武器ID-武器ID-皮肤ID`，但正确格式应为 `武器ID-武器ID-手持特效ID`。
**修复**：投掷物分支单独处理，用皮肤名关键词搜索手持特效ID（920/923/933xxx）。

### Bug 7：1004xxx 检视ID误匹配武器配件
**原因**：`1004` 前缀 7 位，但武器配件 `100412xxx` 也是 9 位且含 `1004` 前缀。
**修复**：加 `maxLen=7` 过滤，ID 长度超过 7 的不算检视动画。

### Bug 8：红装模式显示 430xxx 占位符
**原因**：红装浏览直接展示 430xxx，全部是 `关联=0` 的无效条目。
**修复**：红装模式改为展示 `RED_OUTFITS` 数据（从数据库预筛的 413/423xxx 真实三级皮肤）。

### Bug 9：套装模式搜索显示无效 ID
**原因**：搜索不过滤 `关联=0` 的条目。
**修复**：套装/红装模式下搜索时，跳过 `r==='0'` 的条目。

### Bug 10：多级皮肤（天马神君4个等级）附件匹配失败
**原因**：前缀匹配法（取皮肤ID前N位）在多级皮肤下失效。
**修复**：改为名称双重过滤（同时包含武器名 AND 皮肤系列名的附件才算匹配）。

### Bug 11：AMAP 关键词闭合括号导致截断名称不匹配
**原因**：DB 中附件名称可能截断（如 `M416-天马神君(弹匣` 无闭合 `)`），用 `(弹匣)` 精确匹配失败。
**修复**：AMAP 关键词改为前缀匹配（不含闭合括号），如 `(弹匣` 而非 `(弹匣)`。

## 五、代码格式速查

```
武器皮肤主体：   武器ID-武器ID-皮肤ID
武器附件：       武器ID-槽位ID-配件皮肤ID
挂件：           武器ID-挂件-挂件ID
击打特效：       武器ID-击打特效-特效ID
检视动画：       武器ID-检视-检视动画ID
其他特效：       武器ID-特效ID
投掷物：         武器ID-武器ID-手持特效ID  （仅此一行）
全局换装：       改前ID-改后ID
全局回改：       改后ID-改前ID
局内美化：       部位 = ID
```

## 六、数据更新流程

1. 上传新 `.h` 文件
2. Python 解析 → JSON → gzip → base64 → 替换 HTML 中 `DB` 变量
3. 同步更新：`PERFECT_SKINS`（完美枪皮列表）、`RED_OUTFITS`（真实红装列表）
4. 重新输出 HTML，文件名保持 `pubg_id_query.html`

---

## 七、注意事项

- 所有 JS 必须用 ES5（不用箭头函数、模板字符串），确保低版本 WebView 兼容
- 动态生成的 HTML 中不能用 `onclick="函数名()"` 内联事件，必须用 `addEventListener`
- 浏览结果分页显示（每页80条），支持「加载更多」和「结果内过滤」
- 配置库数据存于内存，刷新页面清空
- 文件最终大小约 880KB，包含全部数据库
